About

Pygate is a HackFS project. Its goal is to be a gateway to the Filecoin network and IPFS technology for Pythonistas. To date, most of the tooling has been geared towards Javascript and Golang developers.

The project is divided into four phases with its own deliverable. Each is intended to serve as a stand-alone component that can be used in other projects.

  1. A Python gRPC interface to the Textile.io Powergate Filecoin API.
  2. A Flask web application that uses the pygate gRCP client to move data to and from the Filecoin network and monitor storage deals.
  3. An Electron desktop application, powered by the Flask app, that allows users to drag-and-drop files to and from the Filecoin network and manage their storage deals.
  4. An Ethereum smart contract, embedded into the desktop app, that manages an escrow account and acts as an agent for storage users to negotiate recurring deals with Filecoin miners to provide them with the desired combination of price, redundancy, and longevity.

The Pygate components are currently in active development and not yet available for release. You can follow progress in our Github repositories, including their wikis. Read more about their requirements below.

Components

gRPC client

  • Release v.0.1.0 of the pygate gRPC client will be used to recreate the same commands/steps from Textile's Powergate tutorial using a Python script that prints to CLI:

    • pow help
pow ffs create
pow ffs addrs list
pow ffs addToHot [filename]
pow ffs push --watch [CID]
pow ffs log [CID]
pow ffs info [CID]
pow ffs config default
pow ffs push [CID] -c [custom_cidconfig.json] -o
pow ffs get [CID]
    • Basic wallet functionality is also critical to a minimal viable product so the gRPC client MVP will include:

      • pow wallet balance
pow wallet new
pow wallet switch

web app

  • Release v.0.1.0 will be a Flask web application that integrates the pygate-gRPC library to make Powergate API calls.
  • Will expose Powergate API and Textile Buckets functionality through a RestX API that can be consumed by the native Flask UI client or other clients such as Slate.
  • Will include a basic file upload dialog to choose files to move to Filecoin.
  • Will allow a user to trigger each of the pygate-gRPC v.0.1.0 commands from a web form and output progress/results to the same webpage.
  • Runs each command as a Celery background task to allow for asynchronous feedback on the webpage (i.e. so it doesn't freeze waiting for Powergate calls to complete).
  • Uses the localnet/testnet wallets that come with a default balance. User management of wallet funding will be address in the desktop app.

desktop app

  • A cross-platform (Windows, Mac OS, Linux) desktop application (e.g. Electron).
  • Allows for drag-n-drop of files and folders from file explorer or Desktop as well as an upload dialog.
  • The files or folders (with nested folders/files) become a Textile Bucket.
  • The app encrypts content by default (option to disable if user wants to publicly share content via IPFS/Filecoin).
  • The user manages their own private keys, including a seed phrase option (i.e. the objective is for a zero-trust architecture).
  • The app manages a Filecoin wallet (possibly in app or in Ethereum contract) to fund the storage deal.
  • The app provides a GUI for the user to set the number of redundant copies on Filecoin network, country preferences (if any), miner allow list (if any), miner deny list (if any).
  • The app notifies the user how long their content will be stored under the terms of the current storage deals.
  • The user has option to toggle number of copies and length of storage to play around with price and duration.
  • The user has option to set frequency of storage deal renewals and to have app automatically renegotiate new storage deals based on their (config preferences) and default storage period (e.g. renegotiate every day, week, month, 2 months, year) to try and arbitrage best rate.
  • The user deposits ETH and/or FIL in smart contract as escrow to fund the initial storage deal and subsequent renewals.
  • The app provides email warnings and pop-up notifications when funding is nearing its end and their content is in danger of being wiped from Filecoin miners (time-period and frequency of warnings is user configurable).
  • The app GUI provides a File Explorer type interface that includes all folders/files that are known to the app with colour coding to indicate whether a file is stored on Filecoin, is about to expire on Filecoin (expiration time-period is a configurable setting), is synced on IPFS, and/or has a local OS copy available.
  • The app GUI provides info on when the last time a proof-of-space-time was run against the file (i.e. serves as a fixity/authenticity check).
  • When user requests file retrieval, the app finds the cheapest retrieval deal amongst the miners in use and delivers files/folders to desktop (or designated folder on OS filesystem).
  • Enhancement: Include IPFS pinning on user-controlled machines and servers or on paid pinning services for redundancy and (hot) availability.
  • Enhancement: Allow for syncing of watched folders on OS (automatic updates to Filecoin Bucket) when a file is added, modified, or deleted in watched folder.

smart contract

  • Serve as an escrow wallet that dispenses ETH and/or FIL to fund Filecoin storage deals.
